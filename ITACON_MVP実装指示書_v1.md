# ITACON MVP 実装指示書 v1

この文書は、ITACON MVPをClaude Codeで実装するための仕様書である。
別添の「ITACON企画書 v1」が思想・背景を含む上位文書だが、**実装はこの指示書のスコープに厳密に従うこと**。この文書にない機能は、企画書に書いてあっても実装しない。

---

# 0. 何を作るか(一文)

プレイヤーが3体のキャラクター(ITACON)から相棒を選び、審査され、憑依し、「どこを狙うか判断 → タイミングを合わせる」の2段操作で共鳴エネルギーを採掘する、**ブラウザで動くシングルプレイのWebゲーム**。

## 検証したい唯一の仮説

> 「ITACONに判断を持ち込み、呼吸を合わせて共鳴を生む」体験は面白いか。そして「どのITACONと組むか」がその面白さを左右するか。

この仮説の検証に寄与しない機能は、すべてスコープ外である。

---

# 1. スコープ

## 1-1. 作るもの

- 画面5つ: 受付(依頼掲示) / マッチング / 採掘(フェーズ1+2) / 精算 / 関係記録の表示
- ITACON 3体(ピート/ジュリアン/パティ)。性格・勘の質・台詞で差別化
- 依頼1種類(共鳴鉱石の採掘)。背景1種類の使い回し
- セッション間で持続するローカルデータ(所持共鳴、各ITACONとのセッション履歴)

## 1-2. 作らないもの(禁止リスト)

以下は企画書に記載があっても**実装しない**。実装中に提案もしない。

- オンライン/マルチプレイヤー要素、サーバー通信
- 3Dグラフィックス、3Dアバター
- 衣装・アイテム購入、贈与システム
- 専属契約システム
- 人間界の密輸市場・共鳴の換金先
- 複数種類のタスク(採掘以外)
- AIによる自由対話(台詞はすべて事前定義のテキスト)
- 4体目以降のキャラクター
- チュートリアル以上の物語・カットシーン
- 音楽・効果音(余裕があれば最後に検討、必須ではない)

# 2. 技術方針

- **単一のWebアプリ**(HTML/CSS/JavaScript)。フレームワークは任意(Vanilla JSで十分な規模。React等を使う場合も軽量に)
- ビルドせずローカルで開けるか、`npm run dev`程度で起動できること
- データ永続化はlocalStorageで十分
- スマホ縦画面でも操作できるレスポンシブ(タップ操作前提)
- グラフィックは当面プレースホルダーでよい(色付き矩形+テキスト)。ただし後から立ち絵画像(PNG)に差し替えられる構造にしておく(3体分の画像スロット)

# 3. ゲームフロー(画面遷移)

```
[受付] → [マッチング] → (拒否されたら受付へ戻る/再挑戦)
                ↓ 承諾
             [採掘 フェーズ1] → [採掘 フェーズ2] → [精算] → [受付]
```

## 3-1. 受付画面

- 依頼が1件表示される: 「第3坑道、共鳴鉱石の採掘。ただし坑道の主がうるさい」程度のフレーバーテキスト
- プレイヤーの所持共鳴エネルギー(累計)を表示
- 「相棒を探す」ボタン → マッチング画面へ

## 3-2. マッチング画面

- ITACON 3体をカードで表示(名前・見た目・一言)
- プレイヤーが1体を選んで「依頼する」
- 選ばれたITACONが**審査**する。結果は以下のいずれか:
  - **承諾**: 採掘へ進む
  - **拒否**: 断り文句を表示し、マッチング画面に戻る(別の相手を選ぶか、同じ相手に再挑戦できる)
- 審査ロジック(シンプルに):
  - 基本承諾率: 70%
  - 同じITACONとのセッション履歴が多いほど承諾率が上がる(1セッションごとに+5%、上限95%)
  - 直前に拒否された相手に連続で申し込むと、その場では確率は変わらないが、**断り文句のバリエーションが変わる**(2回目: 「しつこいって」系の追加台詞)
- 拒否は失敗ではなく、キャラクターを知るイベントとして演出する(コミカルに、冷たくしすぎない)

## 3-3. 採掘画面: フェーズ1(判断)

- 候補地点 **4箇所**(A/B/C/D)をボタンで表示。見た目はほぼ均質にする(反応の強弱を露骨に見せない)
- 真の正解は毎回ランダムに1箇所(非公開)
- 同行しているITACONが**ヒントを一言**発する(例: 「Bのあたり、嫌な予感がする」)
- ヒントの信頼度はキャラごとに異なる(セクション5参照)。信頼度が外れた場合、ヒントはランダムな別の地点を指す
- プレイヤーが1箇所を選んで確定 → フェーズ2へ

## 3-4. 採掘画面: フェーズ2(呼吸合わせ)

- 横に往復するインジケーターバー。中央に成功ゾーン(**全体の20%幅**)
- ゾーンに入った瞬間にタップ/クリック。**3回**繰り返す
- 各タップの精度 = 1 - (ゾーン中心からのズレ距離 / 最大距離)、0〜1にクランプ
- インジケーターの速度は一定(プロトタイプ値: 1フレームあたり2.2%移動、60fps想定)

## 3-5. 精算画面

- 成果計算:
  - `平均精度 = 3回のタップ精度の平均`
  - `正解ボーナス = 選んだ地点が正解なら1.0、不正解なら0.5`
  - `獲得共鳴 = round(平均精度 × 正解ボーナス × 120)`
  - `シンクロ率 = round(平均精度 × 100)%`
- 獲得共鳴とシンクロ率を表示
- ITACONの一言(結果に応じて分岐、セクション5の台詞表を使用):
  - 地点不正解 → 「外れた時の一言」
  - 正解かつシンクロ率80%以上 → 「高シンクロ時の一言」
  - 正解かつシンクロ率50〜79% → 中間の一言
  - 正解かつシンクロ率50%未満 → 「低シンクロ時の一言」
- 「受付に戻る」ボタン

## 3-6. 関係記録

- ITACONごとに保存: セッション回数、累計獲得共鳴、最高シンクロ率
- マッチング画面のカードに表示する(例: 「ピート ○セッション / 最高シンクロ72%」)
- 関係記録は審査の承諾率に反映される(3-2参照)

# 4. コアメカニクスの実装詳細

## 4-1. ヒント生成ロジック

```
正解地点 = random(A..D)
if random() < そのITACONの信頼度:
    ヒント地点 = 正解地点
else:
    ヒント地点 = 正解以外からランダム
ヒント台詞 = キャラごとの言い回しテンプレートに、ヒント地点を埋め込む
```

## 4-2. 数値ベースライン(チューニング前提の初期値)

| パラメータ | 値 |
|---|---|
| 候補地点数 | 4 |
| フェーズ2のタップ回数 | 3 |
| 成功ゾーン幅 | 全体の20% |
| インジケーター速度 | 2.2%/frame |
| 共鳴の係数 | 120 |
| 不正解ペナルティ | ×0.5 |
| 基本承諾率 | 70% |
| セッション履歴による承諾率上昇 | +5%/回(上限95%) |

これらは定数として1箇所にまとめ、簡単に調整できるようにすること。

# 5. キャラクターデータ

## 5-1. ピート(ロボット系)

- 見た目プレースホルダー: 赤系。立ち絵スロット `pete.png`
- ヒント信頼度: **65%(固定)**
- 性格: 新人。元気で自信満々だが当てにならない
- 口調: 元気で丁寧、謎の自信
- 台詞:
  - マッチングカードの一言: 「僕、頑張ります! センサーの精度には自信あるんで!」
  - 承諾: 「はい! 喜んで! 行きましょう!」
  - 拒否: 「今日はちょっと…僕まだ実績少ないんで、無理しない方がいいかもです」
  - 拒否(連続申込時): 「そ、そんなに期待されると逆にプレッシャーというか…」
  - ヒント言い回し: 「{地点}です! 僕のセンサーが反応してます!」
  - 高シンクロ: 「ほら、やっぱり僕の勘、完璧じゃないですか!?」
  - 中間: 「まあまあ、って感じですかね! 次はもっといけます!」
  - 低シンクロ: 「あ、あれ? 息が合わなかったですね…」
  - 外れた時: 「あれ、おかしいな…センサー、今日調子悪いのかも」

## 5-2. ジュリアン(シャーマン系)

- 見た目プレースホルダー: 緑系。立ち絵スロット `julian.png`
- ヒント信頼度: **90%(固定)**
- 性格: 落ちぶれた元・名感応師。無愛想だが勘は本物。生真面目
- 口調: 敬語だが素っ気ない。無駄口を叩かない
- 台詞:
  - マッチングカードの一言: 「……仕事なら、受けます」
  - 承諾: 「いいでしょう。行きます」
  - 拒否: 「今日はやめておきます。あなたと組んでも、いい結果にならない気がするので」
  - 拒否(連続申込時): 「……気が変わるとでも?」
  - ヒント言い回し: 「{地点}。……それだけです」
  - 高シンクロ: 「……悪くなかったですね」
  - 中間: 「可もなく不可もなく、です」
  - 低シンクロ: 「……集中してください。次は」
  - 外れた時: 「……今のは、外れですね。次は外しません。」

## 5-3. パティ(ゾンビ系)

- 見た目プレースホルダー: 青緑系。立ち絵スロット `patty.png`
- ヒント信頼度: **変動型。セッション開始時に40%〜90%からランダムに決定**(内部値。プレイヤーには直接見せないが、台詞の雰囲気でうっすら匂わせてよい)
- 性格: 3体で一番男勝り。乱暴、ガサツ、遠慮がない。酒浸りで稼ぎはすぐ酒に化ける
- 一人称: あたし
- 口調: タメ口でぞんざい、面倒くさそうな言い切り。男口調ではなく雑な女友達のノリ
- 台詞:
  - マッチングカードの一言: 「稼げる話? なら聞くけど」
  - 承諾: 「いいよ、行こ。さっさと稼ご」
  - 拒否: 「今日はナシ。あたし二日酔いでキレそうなんだよね」
  - 拒否(連続申込時): 「しつこいって。明日にしなよ」
  - ヒント言い回し(好調時): 「{地点}だね。あたしの勘、今日冴えてるよ」
  - ヒント言い回し(不調時): 「んー…{地点}? かなあ…ちょっと自信ない」(内部信頼度60%未満の日はこちら)
  - 高シンクロ: 「へえ、やるじゃん。今夜あたしが奢ってやるよ」
  - 中間: 「ま、こんなもんでしょ」
  - 低シンクロ: 「あー最悪、飲みすぎた…次はもっと稼げよ」
  - 外れた時: 「…ごめん、外した。今日のあたしはハズレの日だわ」

# 6. 制約・原則(実装時に守ること)

- **ITACONは全員成人**。台詞・演出・追加要素で年齢が曖昧になる表現を入れない
- 拒絶の演出は「塩対応で笑える」トーン。冷酷・陰湿にしない
- ITACONを一方的な道具として描かない(承諾・拒否・精算時の一言で、常に相手側の意思を感じさせる)
- 台詞はこの指示書の定義済みテキストのみ使用。AI生成・自由対話は入れない
- UIテキストは日本語

---

# 7. 完成の定義(受け入れ条件)

以下がすべて満たされたらMVP完成とする。

1. ブラウザで開き、受付→マッチング→採掘→精算→受付のループが最後まで遊べる
2. 3体それぞれで採掘でき、ヒント信頼度の違い(ピートは当てにならない/ジュリアンは信頼できる/パティは日による)が、複数回遊ぶと体感できる
3. 拒否が発生し、断り文句が表示され、再挑戦できる
4. セッション履歴が保存され、リロード後も保持される。履歴が承諾率と画面表示に反映される
5. スマホの縦画面でタップ操作でプレイできる
6. 数値パラメータ(セクション4-2)が定数としてまとまっており、調整が容易
